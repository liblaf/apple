# Stateful Forward / Inverse API Redesign

## Summary

- Keep one top-level user-facing orchestration API in `liblaf.apple`: `SceneBuilder`, `Scene`, `Forward`, `Inverse`, `Control`, `Loss`, and `ParamRef`.
- Keep backend separation below that layer:
  - `JaxScene` aggregates JAX energies.
  - `WarpScene` aggregates Warp energies.
  - `WarpSceneAdapter` wraps `WarpScene` to the same evaluation API as `JaxScene`.
  - `Scene` combines `JaxScene` and `WarpSceneAdapter`.
- Keep `liblaf.peach` as the numeric engine.
  - `Forward.step()` solves and updates `scene.u_full` in place.
  - `Inverse.solve(x0)` solves and updates both scene params and `scene.u_full` in place.
- V1 supports only fixed/free Dirichlet constraints.
- V1 has no scene-level cell registry, no sliding, no `CombinedControl`, no public `reset()`, no public `commit()`, and no revision/version API.

## Public API And Responsibilities

- `SceneBuilder`
  - Responsibility: assemble a multi-object scene from plain PyVista objects.
  - Methods:
    - `add_object(obj, *, point_ids=None) -> None`
      - Ensures `obj.point_data[GLOBAL_POINT_ID]` exists.
      - Reuses `point_ids` if provided, otherwise reuses existing `GLOBAL_POINT_ID`, otherwise assigns new contiguous ids.
    - `add_dirichlet(obj, *, mask=None, value=None) -> None`
      - Accumulates fixed/free DOF constraints into one global `DirichletSet`.
    - `add_point_set(name, obj, indices) -> None`
      - Stores global point ids derived from `obj.point_data[GLOBAL_POINT_ID][indices]`.
    - `add_energy(energy: JaxEnergy | WarpEnergy) -> None`
      - Registers an already-created energy with the correct backend-local builder.
    - `build() -> Scene`
- `DirichletSet`
  - Responsibility: fixed/free DOF bookkeeping for the one global displacement field.
  - Methods:
    - `get_free(u_full) -> u_free`
    - `to_full(u_free, dirichlet=None) -> u_full`
    - `set_fixed(u_full, values=None) -> u_full`
    - `set_free(u_full, values) -> u_full`
- `ParamRef`
  - Responsibility: stable handle to one actual parameter field on one actual energy instance.
  - Concrete types:
    - `JaxParamRef`
    - `WarpParamRef`
  - Rule: param refs are created from energies, not hand-written from strings.
- `JaxEnergy` / `WarpEnergy`
  - Responsibility: one energy term plus its own backend-local parameter storage and energy state.
  - Public additions:
    - `param_names() -> tuple[str, ...]`
    - `param(name: str) -> ParamRef`
- `Scene`
  - Responsibility: top-level mutable runtime object.
  - Owns:
    - accepted `u_full`
    - `DirichletSet`
    - `JaxScene`
    - `WarpSceneAdapter`
    - named point sets
  - Accepted-state methods:
    - `u_full() -> Array`
    - `u_free() -> Array`
    - `set_u_full(u) -> None`
    - `set_u_free(u_free) -> None`
    - `reset_u() -> None`
    - `param(ref: ParamRef) -> Array`
    - `update_param(ref: ParamRef, value) -> None`
    - `update_params(mapping: dict[ParamRef, Array]) -> None`
    - `point_set(name) -> Array`
  - Pure forward-evaluation methods:
    - `init_forward_state() -> ForwardState`
    - `update_state(state: ForwardState, u_full) -> ForwardState`
    - `fun(state: ForwardState) -> Scalar`
    - `grad(state: ForwardState) -> Array`
    - `hess_diag(state: ForwardState) -> Array`
    - `hess_prod(state: ForwardState, v_full) -> Array`
    - `mixed_hess_prod(state: ForwardState, p_full) -> dict[ParamRef, Array]`
- `ForwardState`
  - Responsibility: pure per-iterate forward evaluation state used inside `liblaf.peach`.
  - Fields:
    - `u_full`
    - `jax`
    - `warp`
- `InverseState`
  - Responsibility: pure per-iterate inverse evaluation state used inside `liblaf.peach`.
  - Fields:
    - `params`
    - `control_state`
    - `forward_state`
    - ordered parameter arrays needed by losses
  - Methods:
    - `u_full() -> Array`
      - Returns `forward_state.u_full`.
    - `param(ref: ParamRef) -> Array`
- `Loss`
  - Responsibility: pure scalar objective term over `InverseState`.
  - Methods:
    - `required_params() -> tuple[ParamRef, ...]`
    - `fun(state: InverseState) -> Scalar`
  - Built-ins:
    - `PointToPointLoss(indices, target, weights=None)`
    - `UniformFieldLoss(param, indices=None)`
    - `SmoothFieldLoss(param, pairs)`
- `Control`
  - Responsibility: flat-vector boundary between optimizers and scene parameters.
  - Methods:
    - `get_vector(scene: Scene) -> Vector`
    - `update_scene(scene: Scene, params: Vector) -> object`
    - `get_vector_grad(scene: Scene, control_state: object, param_grads: dict[ParamRef, Array]) -> Vector`
- `FieldControl`
  - Responsibility: V1’s only built-in control.
  - Fields:
    - `param: ParamRef`
    - `indices: Array | None`
  - Behavior:
    - `get_vector` flattens the whole parameter field or the indexed subset.
    - `update_scene` writes the flat vector back into the same field or subset.
    - `get_vector_grad` selects the same field or subset from `param_grads` and flattens it.
- `Forward`
  - Responsibility: stateful equilibrium solver over the current scene parameters.
  - Fields:
    - `scene`
    - `optimizer: liblaf.peach.optim.Optimizer`
    - `state: ForwardState | None`
    - `last_forward_solution`
  - Methods:
    - `step(callback=None, *, logging=True) -> peach.optim.Optimizer.Solution`
- `Inverse`
  - Responsibility: stateful inverse solver over scene parameters.
  - Fields:
    - `forward`
    - `control`
    - `losses`
    - `optimizer: liblaf.peach.optim.Optimizer`
    - `adjoint_solver: liblaf.peach.linalg.LinearSolver`
    - `state: InverseState | None`
    - `last_inverse_solution`
    - `last_adjoint_solution`
  - Methods:
    - `update(params: Vector) -> InverseState`
    - `fun(state: InverseState) -> Scalar`
    - `grad(state: InverseState) -> Vector`
    - `value_and_grad(state: InverseState) -> tuple[Scalar, Vector]`
    - `solve(x0=None, callback=None) -> peach.optim.Optimizer.Solution`

## Backend Scenes

- `JaxScene`
  - Responsibility: aggregate all JAX energies and expose one pure evaluation API over the global displacement field.
  - Methods:
    - `init_forward_state(u_full) -> JaxForwardState`
    - `update_state(state, u_full) -> JaxForwardState`
    - `fun(state) -> Scalar`
    - `grad(state) -> Array`
    - `hess_diag(state) -> Array`
    - `hess_prod(state, v_full) -> Array`
    - `mixed_hess_prod(state, p_full) -> dict[JaxParamRef, Array]`
    - `param(ref: JaxParamRef) -> Array`
    - `update_param(ref: JaxParamRef, value) -> None`
- `WarpScene`
  - Responsibility: native Warp aggregation of Warp energies, Warp parameter storage, and Warp-side temporary buffers.
- `WarpSceneAdapter`
  - Responsibility: expose the same API shape as `JaxScene` on top of `WarpScene`.
  - Methods:
    - `init_forward_state(u_full) -> WarpForwardState`
    - `update_state(state, u_full) -> WarpForwardState`
    - `fun(state) -> Scalar`
    - `grad(state) -> Array`
    - `hess_diag(state) -> Array`
    - `hess_prod(state, v_full) -> Array`
    - `mixed_hess_prod(state, p_full) -> dict[WarpParamRef, Array]`
    - `param(ref: WarpParamRef) -> Array`
    - `update_param(ref: WarpParamRef, value) -> None`
  - Rule: all Warp-to-JAX bridging lives here.

## Solve Flow

- `_ForwardObjective`
  - Responsibility: adapt `Scene` to the `liblaf.peach` forward objective interface on free DOFs.
  - Methods:
    - `update(state, u_free) -> ForwardState`
      - Converts `u_free` to full displacement with `scene.dirichlet.to_full(...)`.
      - Calls `scene.update_state(...)`.
    - `fun(state) -> Scalar`
      - Returns `scene.fun(state)`.
    - `grad(state) -> Vector`
      - Returns `scene.dirichlet.get_free(scene.grad(state))`.
    - `hess_diag(state) -> Vector`
      - Returns `scene.dirichlet.get_free(scene.hess_diag(state))`.
    - `hess_prod(state, p_free) -> Vector`
      - Lifts `p_free` to full coordinates with zero Dirichlet entries, calls `scene.hess_prod(...)`, then restricts back to free DOFs.
- `Forward.step()`
  - Uses current `scene.u_free()` as the optimizer initial guess.
  - Initializes pure state with `scene.init_forward_state()`.
  - Calls `optimizer.minimize(...)` with `_ForwardObjective`.
  - Stores returned `ForwardState` in `self.state`.
  - Stores returned optimizer solution in `last_forward_solution`.
  - Updates accepted `scene.u_full` from the accepted optimizer parameters.
- `_InverseObjective`
  - Responsibility: adapt `Inverse` to the `liblaf.peach` inverse objective interface on the flat control vector.
  - Methods:
    - `update(_state, params) -> InverseState`
      - Returns `inverse.update(params)`.
    - `fun(state) -> Scalar`
      - Returns `inverse.fun(state)`.
    - `grad(state) -> Vector`
      - Returns `inverse.grad(state)`.
    - `value_and_grad(state) -> tuple[Scalar, Vector]`
      - Returns `inverse.value_and_grad(state)`.
- `Inverse.update(params)`
  - Calls `control.update_scene(scene, params)`.
  - Calls `forward.step()`.
  - Gathers all param refs required by the control and losses.
  - Reads those param arrays from the accepted scene.
  - Builds and returns `InverseState`.
  - Stores it in `self.state`.
- `Inverse.fun(state)`
  - Returns the sum of `loss.fun(state)` for all losses.
- `Inverse.value_and_grad(state)`
  - Uses JAX to differentiate the summed loss with respect to:
    - `state.forward_state.u_full`
    - the ordered parameter arrays stored in `state`
  - Builds the adjoint system from `forward.state` and direct `dL/du`.
  - Solves the reduced free-DOF adjoint system with `adjoint_solver.solve(...)`.
  - Stores that result in `last_adjoint_solution`.
  - Lifts the adjoint vector to full coordinates with `scene.dirichlet.to_full(..., dirichlet=0.0)`.
  - Calls `scene.mixed_hess_prod(state.forward_state, lambda_full)` to get implicit parameter gradients for all differentiable energy params.
  - Adds direct and implicit parameter gradients by `ParamRef`.
  - Returns the scalar loss and `control.get_vector_grad(scene, state.control_state, param_grads)`.
- `Inverse.grad(state)`
  - Calls `value_and_grad(state)` and returns only the gradient.
- `Inverse.solve(x0=None, callback=None)`
  - Defaults `x0` to `control.get_vector(scene)`.
  - Calls `optimizer.minimize(...)` with `_InverseObjective`.
  - Stores returned `InverseState` in `self.state`.
  - Stores the optimizer solution in `last_inverse_solution`.
  - Re-applies the accepted optimizer vector through `control.update_scene(...)`.
  - Runs one final `forward.step()` so accepted scene params and accepted `scene.u_full` are synchronized.

## Example Usage

```python
builder = SceneBuilder(dim=3)

builder.add_object(volume)
builder.add_object(surface, point_ids=surface.point_data[GLOBAL_POINT_ID])

builder.add_dirichlet(volume)
builder.add_point_set("surface", surface, np.arange(surface.n_points))

fat = WarpArap.from_pyvista(volume, name="fat")
muscle = WarpArapMuscle.from_pyvista(
    volume,
    name="muscle",
    requires_grad=("activation",),
)
surface_spring = JaxMassSpring.from_pyvista(surface, name="surface_spring")

builder.add_energy(fat)
builder.add_energy(muscle)
builder.add_energy(surface_spring)

scene = builder.build()

activation = muscle.param("activation")
surface_points = scene.point_set("surface")
muscle_cells = np.flatnonzero(volume.cell_data["MuscleFraction"] > 1e-3)

forward = Forward(scene=scene, optimizer=PNCG(...))
inverse = Inverse(
    forward=forward,
    control=FieldControl(activation, indices=muscle_cells),
    losses=[
        PointToPointLoss(indices=surface_points, target=target_u_surface),
        UniformFieldLoss(activation, indices=muscle_cells),
    ],
    optimizer=Optax(...),
    adjoint_solver=FallbackSolver(),
)

inverse.solve()

u_surface = scene.u_full()[surface_points]
activation_full = scene.param(activation)
```

## Assumptions And Explicit Omissions

- PyVista objects are the object handles. There is no `SceneObjectRef`.
- Energy instances are the energy handles. There is no `EnergyRef`.
- `ForwardState` is the pure forward-evaluation state. `InverseState` replaces the old snapshot concept.
- There is no public `Scene.commit(state)` because accepted scene state is updated directly by `Forward.step()` and `Inverse.solve()`.
- There is no public revision or version tracking API. `Forward.step()` always initializes a fresh pure `ForwardState` from the current accepted scene state.
- There is no public `Forward.reset()`.
- V1 keeps only one built-in control: `FieldControl`.
- `Scene.mixed_hess_prod(...)` returns all differentiable energy params owned by the registered energies. It does not take a param filter argument.
- V1 supports only fixed/free Dirichlet constraints and one global displacement field `u`.
- V1 does not include sliding, scene-level cell sets, or combined control composition helpers.
